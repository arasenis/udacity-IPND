Lesson One: Introduction to "Serious" Programming

Computer

Most machines that we use today are designed to do a few things. Unless you physcially change this things they will only be able to do the thing they are designed for.

However, computers are not an example of that. They are programmed to do whatever we want. To tell the computer what to do, we need to give some instructions to it.

Computer Program

A program is a precise sequence of steps that a computer can follow to do something useful. Web browsers, games, mobile apps, and simple print statements are all examples of computer programs.

Programming Language
A programming language is what programmers use to tell a computer what to do. Python and Objective-C are examples of programming languages.
Python
Python is a programming language. When you write Python code and press "Run", a Python Interpreter converts the code you wrote as a set of instructions that the computer itself can understand and execute.
Grammar

Just like human languages have grammars, programming languages do too. A grammar is a specification of what is "correct" and what is "incorrect."

In languages like English, people can often make sense of sentences that aren't technically "correct," but computers aren't smart enough to do this. This means we have to write code that is exactly "correct" according to the Python interpreter, otherwise our code won't run.

Python Expressions

A Python "expression" is a legal Python statement. For example: print 1 + 1is a valid expression, but print 1 + (without a number at the end) is not.

Lesson Three: Lesson 3: Input -> Function -> Output

What is a function?

A function is something that takes input, does something to that input, and then produces output.

For example, a function named cube might take a number and give the numbers cube out.

What is the difference between making and using a function?

Functions are made by starting a line of code with the keyword def and then giving a function name followed by the function parameters in parentheses. These parameters will eventually be replaced by actual values when the function is used (called).

In the "body" of the function, we write the code that specifies what to do with the input parameters. For example the following code could be the definition of a function called cube:


def cube(s):
answer = s * s * s
return answer

To use a function, we write the name of the function followed by the value(s) we want to give it in parentheses. Like this:

print cube(4) >>>64

How do functions help programmers avoid repetition?

Functions give programmers the chance to create and reuse it forever. When you define a function once, you never have to define it again.

What happens if a function doesn't have a return statement?
The return statement tells the function what it should produce. If a function doesn't have a return function it won't know what to produce so you will get an error.

What is OOP

Also called OOP, Object-oriented programming (OOP) is a programming paradigm that represents the concept of "objects" that have data fields and associated procedures. Objects, which are usually instances of classes, interact with one another to design applications and computer programs. Main idea behind OOP is to manage the complexity of software systems. Applications build over OOP are easier to maintain, have more reusable components, and are more scalable.

How Classes and Objects are defined in OOP?

If suppose, you are creating a application for automobile industry then you can think of a computer as a class and a particular type of computer as an object which will be used as instance of computer class.

Data Member : computer processor, CPU , max resolution, colour etc.. Methods: GetData(), GetCPU(), Use()..

Advantages of OOP

Improved software-development productivity: Object-oriented programming is modular, as it provides separation of duties in object-based program development. It is also extensible, as objects can be extended to include new attributes and behaviors. Objects can also be reused within an across applications. Because of these three factors – modularity, extensibility, and reusability – object-oriented programming provides improved software-development productivity over traditional procedure-based programming techniques.

Improved software maintainability: For the reasons mentioned above, objectoriented software is also easier to maintain. Since the design is modular, part of the system can be updated in case of issues without a need to make large-scale changes.

3. Faster development: Reuse enables faster development. Object-oriented programming languages come with rich libraries of objects, and code developed during projects is also reusable in future projects.

OOP Important Vocabularies

Object:An object can be a variable, a data structure, or a function. In the class-based object-oriented programming paradigm, "object" refers to a particular instance of a class where the object can be a combination of variables, functions, and data structures.

Class:A class is a blueprint or template or set of instructions to build a specific type of object. Every object is built from a class. Each class should be designed and programmed to accomplish one, and only one, thing. Because each class is designed to have only a single responsibility, many classes are used to build an entire application.

Instance:An instance is a specific object built from a specific class. It is assigned to a reference variable that is used to access all of the instance's properties and methods. When you make a new instance the process is called instantiation and is typically done using the new keyword.

Method Once you have created objects, you want them to be able to do something. This is where methods come in. A method in object-oriented programming is a procedure associated with a class. A method defines the behavior of the objects that are created from the class. Another way to say this is that a method is an action that an object is able to perform. The association between method and class is called binding. Consider the example of an object of the type 'person,' created using the person class. Methods associated with this class could consist of things like walking and driving. Methods are sometimes confused with functions, but they are distinct.

Library:A class library is a pre-coded object-oriented programming (OOP) template collection. Both network and desktop applications use class libraries. Class libraries contain code for graphical user interface (GUI) elements such as buttons, icons, scroll bars and windows as well as other non-GUI components.

All OOP languages have class libraries. Languages with .NET framework use Base Class Library (BCL). Java languages use Java Class Library (JCL).

Module:The reusable components should be as "modular" as possible. A module is a component of a larger system that interacts with the rest of the system in a simple, well-defined, straightforward manner. The idea is that a module can be "plugged into" a system. The details of what goes on inside the module are not important to the system as a whole, as long as the module fulfills its assigned role correctly. This is called information hiding, and it is one of the most important principles of software engineering.

Abstract Thinking

Objects in programming are designed to be broken down to the least common thing similar in nature used in order to build upon it with other objects to create new objects. In this way, each object can be reused and enhanced to suite the needs of other applications.

Classes are used to identify objects throughout all Object Oriented Programming based languages. This is noted in JavaScript and HTML in such a way that CSS can be used to identify such classes and manipulate them based on the rules defined in the style guide.

In simple terms, Object Oriented Programming is the science of looking at daily life items (objects) and breaks them down into the simplest reusable item to create classes based on that items description of how it acts (methods) and is described (variables).

Networks
A Network consists of a group of computers, servers or other entities that can communicate with each other even though they are not directly connected together. Routers and switches connect the entities together in a transparent infrastructure which allows the Client (web browser) and server (web server) to interact via protocols such as HTTP. HTTP helps facilitate the GET and POST functions. GET is used to retrieve data from a remote host or server and POST sends information to be saved or processed on the server.
Servers

Servers can come in many forms. They can be a local development server running on a virtual machine on your computer all the way up to a cloud based highly scalable platform like the Google or AT and T PaaS. Servers respond to GET request and store and/or process POST requests from the client. Web Servers are a place to make your web application available to the web and offer many benefits to hosting a web application yourself. Web Servers are connected using high speed data connections and are usually strategically located in areas where a major hub for internet traffic is located. Some Servers Providers also offer replication so your app can be located on servers all over the world; this ensures your clients will be able to access your app with less latency.

An example of a URL: http://www.hey.com/

How does a server work? Servers wait for requests - when a request is being made - the server processes the request and using a protocol it delievers the data to the requester. Let's say that we're surfing the network using a computer - when we type a url or clicking a link - our web broswer is sending a request to the servers - and assuming everything is valid (both on server and client side) - the servers response with what we requestsed.

Validation
Validation of the user input data is a necessity, because it combats malicious input that can make your web app vulnerable to a hacker taking down your site or corrupting your database. Validation checks the user input against what you are expecting the user to put in the fields. If the data does not fit the parameters set in the validation code an error is returned to the user to correct the input information. Validation can be performed on the client side in the web browser, on the server side or both the client and server.
Templates
Templates are used to separate different types of code. It helps keeps HTML out of your Python code, which make the code more readable and easier to modify. Template Inheritance is used to minimize the overall code by establishing a base template for repeated elements in the web app, such as the header and footer that can be the same across all pages. Python modules provide abstraction from the behind the scenes complicated code that someone has already perfected and allows you to concentrate on other aspects of the application.
